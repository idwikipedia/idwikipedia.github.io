2. Mengapa protokol routing tipe link state lebih baik dibanding kan dengan distance vector? Jelaskan


Jawaban:

Protokol routing tipe link state dan distance vector adalah dua pendekatan yang berbeda dalam perutean jaringan komputer. Berikut adalah beberapa alasan mengapa protokol routing tipe link state dianggap lebih baik daripada distance vector:

1. Informasi Topologi yang Lebih Akurat: Protokol routing tipe link state mempertahankan database yang lengkap tentang topologi jaringan, termasuk status dan metrik dari setiap link di jaringan. Dalam protokol distance vector, setiap router hanya memiliki informasi tentang jarak ke tujuan tertentu. Dengan informasi topologi yang lebih akurat, protokol link state dapat membuat keputusan perutean yang lebih cerdas dan memilih jalur terbaik berdasarkan kondisi aktual jaringan.

2. Konvergensi yang Lebih Cepat: Dalam protokol routing tipe link state, setiap router memiliki pemahaman yang sama tentang topologi jaringan. Ketika terjadi perubahan dalam jaringan, seperti kegagalan link atau perubahan metrik, setiap router secara langsung mendapatkan informasi tentang perubahan tersebut. Dalam protokol distance vector, diperlukan waktu yang lebih lama untuk menyebarkan informasi perubahan melalui iterasi terus-menerus antara router. Oleh karena itu, protokol link state cenderung memiliki waktu konvergensi yang lebih cepat.

3. Skalabilitas yang Lebih Baik: Protokol link state mampu mengatasi masalah skala dengan lebih baik daripada protokol distance vector. Dalam protokol link state, setiap router hanya perlu menyimpan informasi tentang tetangga langsungnya dan bukan tentang seluruh jaringan. Ini membuat protokol link state lebih efisien dalam penggunaan sumber daya dan mampu mengatasi jaringan yang lebih besar dengan jumlah router yang lebih banyak.

4. Resiliensi Terhadap Routing Loop: Protokol link state menggunakan algoritma Dijkstra untuk menghitung jalur terbaik. Algoritma ini dapat mencegah terjadinya routing loop, yaitu situasi di mana paket data terus-menerus beredar dalam loop tak terbatas antara beberapa router. Di sisi lain, protokol distance vector lebih rentan terhadap routing loop karena informasi perutean hanya diketahui secara terbatas oleh setiap router.

Meskipun protokol routing tipe link state memiliki keunggulan di atas, penting untuk mencatat bahwa setiap protokol routing memiliki kelebihan dan kelemahan tergantung pada kebutuhan dan karakteristik jaringan tertentu. Pemilihan protokol routing harus mempertimbangkan faktor-faktor seperti ukuran jaringan, kompleksitas topologi, dan persyaratan kinerja yang diinginkan.

3. apa bedanya protokol routing distance vector dan link state?


Perbedaan protokol Routing Distance Vector dan Link State adalah sebagai berikut

Berdasarkan algoritma

Distance Vector Roputing menggunakan algoritma Bellman Ford. Sedangkan Link State Routing menggunakan algoritma Dijsktra

Berdasarkan jaringan

Pada Distance Vector Routing, informasi topologi didapatkan dari point of view tetangga. Sedangkan pada Link State Routing, keseluruhan informasi didapatkan pada topologi jaringan

Berdasarkan kalkulasi path

Pada Distance Vector Routing, kalkulasi berdasarkan pada jumlah hop terkecil. Sedangkan pada Link State Routing, kalkulasi berdasarkan pada cost

Berdasarkan update

Pada Distance Vector Routing, menggunakan Full Routing Table. Sedangkan pada Link State Routing, menggunakan Link State Update

Berdasarkan frekuensi update

Pada Distance Vector Routing, update dilakukan secara periodik. Sedangkan pada Link State Routing, update dilakukan jika terdapat trigger

Berdasarkan CPU dan Memori

Pada Distance Vector Routing, CPU dan MEmori sedikit digunakan. Sedangkan pada Link State Routing, CPU dan Memori digunakan secara intensif

Berdasarkan kemudahan

Distance Vector Routing mudah dilakukan. Sedangkan pada Link State Routing, membutuhkan administrator jaringan yang ahli

Berdasarkan waktu konvergensi

Pada Distance Vector Routing, waktu konvergensi cenderung moderat. Sedangkan pada Link State Routing, waktu konvergensi cenderung cepat

Berdasarkan struktur hierarki

Distance Vector Routing memiliki struktur hierarki. Sedangkan Link State Routing tidak memiliki struktur hierarki

Berdasarkan node intermediate

Distance Vector Routing tidak memiliki node intermediate. Sedangkan Link State Routing memiliki node intermediate

Mapel : TIK

Kode : 9.11.2

Kategori : Dasar-Dasar Sistem Jaringan Internet dan Intranet

Kata kunci : Distance Vector Routing, Link State Routing

7. Klasifikasi routing protokol dibedakan menjadi beberapa jenis, yang bukan termasuk dari klasifikasi routing protokol adalah ....A. Path VectorB. Link State atau Shortest Path FirstC. HybridD. Distance VectorE. Multi Dimensi​


E. Multi Dimensi

Dalam jaringan komputer, routing protokol adalah sekumpulan aturan yang mengatur bagaimana paket data di-routing atau diteruskan melalui jaringan. Routing protokol dapat diklasifikasikan ke dalam beberapa jenis, di antaranya adalah:

Distance Vector: routing protokol yang menggunakan tabel jarak untuk menentukan rute terbaik ke tujuan.Link State atau Shortest Path First (SPF): routing protokol yang menggunakan informasi keadaan (state) dari semua tautan (link) di jaringan untuk menentukan rute terbaik ke tujuan.Path Vector: routing protokol yang menggunakan informasi vektor jalur (path vector) untuk menentukan rute terbaik ke tujuan.Hybrid: routing protokol yang menggabungkan karakteristik dari beberapa jenis routing protokol lainnya.

Dari pilihan jawaban di atas, hanya pilihan E yang tidak termasuk dalam klasifikasi routing protokol, yaitu Multi Dimensi

10. Jelaskan pengertian asas keagamaan dalam asas-asas keprotokolan​


Jawaban:

Dalam konteks keprotokolan, asas keagamaan merujuk pada prinsip atau nilai-nilai yang berkaitan dengan agama dalam pelaksanaan protokol atau tatacara resmi di lingkungan keagamaan. Asas keagamaan ini diterapkan untuk memastikan bahwa pelaksanaan kegiatan protokoler di wilayah keagamaan disesuaikan dengan nilai-nilai agama yang dijunjung tinggi oleh masyarakat agama tertentu.Beberapa hal yang berkaitan dengan asas keagamaan dalam asas-asas keprotokolan adalah:1. Penghargaan terhadap kepercayaan dan praktik agama: Asas keagamaan menegaskan pentingnya menghormati kepercayaan dan praktik agama yang berbeda-beda. Dalam kegiatan protokoler yang melibatkan berbagai agama, asas ini memastikan bahwa semua pihak mendapatkan perlakuan yang adil dan menghormati hak-hak agama serta kebebasan beragama.2. Penyesuaian waktu dan tempat: Asas keagamaan juga melibatkan penyesuaian waktu dan tempat dalam pelaksanaan protokol. Misalnya, dengan memperhatikan kalender agama atau hari libur keagamaan, protokol dapat diatur agar tidak bertentangan dengan kegiatan atau larangan agama tertentu. Demikian pula, tempat yang digunakan untuk kegiatan protokoler di lingkungan keagamaan haruslah mempertimbangkan kebutuhan dan aturan agama yang berlaku.3. Etika dan tata krama agama: Asas keagamaan juga mencakup penghormatan terhadap etika dan tata krama agama. Ini termasuk melibatkan pemimpin agama atau ulama dalam acara-acara resmi, mengikutsertakan upacara keagamaan atau ritual dalam protokol, atau mempertimbangkan keberagaman denominasi agama dalam acara resmi.4. Pembatasan dan ketentuan agama: Asas keagamaan juga mengakui adanya pembatasan dan ketentuan agama tertentu yang harus diperhatikan dalam protokol. Misalnya, dalam hal pakaian atau penutup kepala yang wajib, pemisahan gender dalam beberapa kegiatan, atau larangan makanan tertentu dalam proses persiapan makanan yang akan disajikan.Penerapan asas keagamaan dalam asas-asas keprotokolan penting untuk menciptakan lingkungan yang inklusif, menghormati dan mengakui perbedaan agama, dan mampu memberikan rasa nyaman serta keadilan bagi semua pihak yang terlibat dalam kegiatan protokoler di lingkungan keagamaan.

Jawaban:

Dalam konteks asas-asas keprotokolan, asas keagamaan merujuk pada prinsip-prinsip yang mengakui dan menghormati keberagaman agama serta memberikan perlakuan yang adil dan setara terhadap individu berdasarkan keyakinan keagamaan mereka. Asas keagamaan biasanya diterapkan dalam konteks protokol yang melibatkan pertemuan, acara resmi, atau kegiatan lain yang melibatkan orang-orang dengan latar belakang agama yang berbeda.

Beberapa prinsip asas keagamaan dalam keprotokolan antara lain:

Kebebasan beragama: Menghormati hak setiap individu untuk memeluk agama atau keyakinan apa pun dan berlatih secara bebas tanpa adanya diskriminasi atau intimidasi.

Keadilan dalam perlakuan: Memastikan bahwa setiap orang, terlepas dari agama atau keyakinan mereka, diperlakukan secara adil dan setara dalam segala aspek pembuatan keputusan atau pelaksanaan protokol.

Toleransi dan penghormatan: Menghargai perbedaan agama dan keyakinan serta memperluas pengertian dan penghargaan terhadap keberagaman tersebut.

Menjaga netralitas: Dalam beberapa situasi, terutama dalam konteks resmi atau pemerintahan, asas keprotokolan dapat mewajibkan netralitas dalam hal agama untuk memastikan bahwa tidak ada keberpihakan atau penindasan terhadap kelompok atau individu berdasarkan agama mereka.

Mengakomodasi kebutuhan agama: Mencakup tindakan atau prosedur untuk memfasilitasi praktik dan kebutuhan agama yang penting bagi individu atau kelompok dalam suatu kegiatan atau acara protokol.

Penerapan asas keagamaan dalam asas-asas keprotokolan bertujuan untuk menciptakan lingkungan yang inklusif, saling menghormati, dan memfasilitasi partisipasi seluruh individu terlepas dari latar belakang agama mereka.

Karakteristik dari distance vector protocol meliputi:

A. Memilih rute berdasarkan jarak terpendek dari suatu node ke node lainnya

B. Biasanya menggunakan algoritma Bellman-Ford untuk menghitung jarak terpendek

26. tuliskan cara kerja protokol netware


Like TCP/IP, Novell provides a suite of protocols developed specifically for NetWare. The five main protocols used by NetWare are:Media Access Protocol.Internetwork Packet Exchange/Sequenced Packet Exchange (IPX/SPX).Routing Information Protocol (RIP).Service Advertising Protocol (SAP).NetWare Core Protocol (NCP).Because these protocols were defined well before the finalization of the OSI reference model, they do not exactly match OSI. Figure 6.5 provides mapping of the NetWare protocols to the OSI reference model. In actuality, no direct correlation to the layer boundaries of the two architectures exists. These protocols follow an enveloping pattern. More specifically, the upper-lever protocols (NCP, SAP, and RIP) are enveloped by IPX/SPX. 27. 2. Ketika anda membangun jaringan berbasis protocol routing distance vector, anda harus beberapa ciri atau karakteristiknyan. Di antara karakteristik distance vector yang tidak tepat adalah memahami C. Distance D. Classless protocol A. Vector B. Classfull routing E. Kurang sesuai diterapkan pada jaringan berskala besar.​


Jawaban:

Jawaban yang tidak tepat dalam konteks karakteristik jaringan berbasis protocol routing distance vector adalah B. Classfull routing.

Karakteristik yang seharusnya dipahami dalam jaringan berbasis protocol routing distance vector adalah:

A. Vector: Routing distance vector menggunakan informasi vector (vektor) untuk menggambarkan jarak dan arah ke tujuan dalam jaringan. Setiap node (router) mengirimkan vektor yang berisi informasi tentang jaraknya ke tujuan yang diketahui.

C. Distance: Distance vector routing menghitung jarak ke tujuan berdasarkan metrik tertentu, seperti jumlah hop, biaya, atau delay. Informasi jarak ini digunakan untuk membuat keputusan routing.

D. Classless protocol: Distance vector routing dapat mendukung penggunaan subnetting dan VLSM (Variable Length Subnet Masking). Oleh karena itu, protokol routing distance vector tidak terbatas pada penggunaan kelas (classful), di mana subnetting tidak dapat dilakukan dengan fleksibilitas.

E. Kurang sesuai diterapkan pada jaringan berskala besar: Karakteristik ini benar. Protokol routing distance vector, seperti RIP (Routing Information Protocol), cenderung kurang efisien dan kurang cocok untuk jaringan yang sangat besar dengan banyak rute. Hal ini disebabkan oleh keterbatasan dalam propagasi informasi routing, konvergensi yang lambat, dan kemungkinan terjadinya loop routing.

Jadi, jawaban yang tidak tepat adalah B. Classfull routing.

30. bagaimana cara kerja protokol routing bgp


Jawaban:

Routing protokol BGP baru dapat dikatakan bekerja pada sebuah router jika sudah terbentuk sesi komunikasi dengan router tetangganya yang juga menjalankan BGP. Sesi komunikasi ini adalah berupa komunikasi dengan protokol TCP dengan nomor port 179. Setelah terjalin komunikasi ini, maka kedua buah router BGP dapat saling bertukar informasi rute.

Untuk berhasil menjalin komunikasi dengan router tetangganya sampai dapat saling bertukar informasi routing, ada beberapa hal yang perlu diperhatikan:

1. Kedua buah router telah dikonfigurasi dengan benar dan siap menjalankan routing protokol BGP.

2. Koneksi antarkedua buah router telah terbentuk dengan baik tanpa adanya gangguan pada media koneksinya.

3. Pastikan paket-paket pesan BGP yang bertugas membentuk sesi BGP dengan router tetangganya dapat samp dengan baik ke tujuannya.

4. Pastikan kedua buah router BGP tidak melakukan pemblokiran port komunikasi TCP 179.

5. Pastikan kedua buah router tidak kehabisan resource saat sesi BGP sudah terbentuk dan berjalan.

Setelah semuanya berjalan dengan baik, maka sebuah sesi BGP dapat bekerja dengan baik pada router Anda.
