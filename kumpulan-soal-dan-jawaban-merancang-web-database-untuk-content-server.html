15. SOAL PRODUKTIFI KELAS XI TKJ (A-D) 1. Jelaskan yang di maksud dengan web server, dan berikan 2 contoh serta jelaskan web server yang banyak digunakan pada saat ini! 2. Sebutkan 3 contoh aplikasi Web Browser dan jelaskan masing-masing 3 contoh? 3. sebutkan 3 contoh bahasa pemrograman yang digunakan untuk membuat sebuah web. Kemudian jelaskan cara kerja sebuah web server! 4. Jelaskan apa yang dimaksud dengan DNS Server! 5. Sebutkan dan jelaskan fungsi dari DNS Server!​


Jawaban:

1. Web server adalah sebuah program atau software yang berfungsi untuk menerima dan memproses permintaan dari browser web untuk menampilkan halaman web.

Dua contoh web server yang banyak digunakan saat ini adalah Apache dan Nginx. Apache adalah web server open-source yang sangat populer dan digunakan oleh sebagian besar situs web di seluruh dunia. Nginx adalah web server open-source yang dirancang untuk menangani lalu lintas web dengan efisien pada server modern yang sibuk.

2. Tiga contoh aplikasi web browser yang populer adalah:

Google Chrome: aplikasi web browser yang dikembangkan oleh Google. Fiturnya meliputi tab browsing, penjelajahan pribadi, dan integrasi dengan Google Drive dan aplikasi Google lainnya.Mozilla Firefox: aplikasi web browser open-source yang populer dengan fitur-fitur seperti tab browsing, penjelajahan pribadi, dan dukungan untuk add-on dan ekstensi.Microsoft Edge: aplikasi web browser yang dikembangkan oleh Microsoft. Fiturnya meliputi tab browsing, integrasi dengan Microsoft Office dan OneDrive, dan mode penghematan baterai untuk perangkat mobile.

3. Tiga contoh bahasa pemrograman yang digunakan untuk membuat web adalah:

HTML (Hypertext Markup Language): bahasa markah yang digunakan untuk membuat struktur dan konten halaman web.CSS (Cascading Style Sheets): bahasa stylesheet yang digunakan untuk mengatur tampilan dan format halaman web.JavaScript: bahasa scripting yang digunakan untuk membuat interaksi dan efek di halaman web.

Web server bekerja dengan menerima permintaan dari browser web melalui protokol HTTP (Hypertext Transfer Protocol) atau HTTPS (Hypertext Transfer Protocol Secure). Setelah menerima permintaan, server memproses permintaan tersebut dan mengirimkan kembali halaman web yang diminta ke browser.

4. DNS Server (Domain Name System Server) adalah sebuah server yang berfungsi untuk mengubah nama domain (seperti google.com) menjadi alamat IP yang sesuai (seperti 172.217.168.46) sehingga dapat diakses oleh komputer melalui jaringan internet.

5. Fungsi dari DNS Server adalah sebagai berikut:

Menerjemahkan nama domain menjadi alamat IP yang sesuai.Mengelola cache DNS untuk meningkatkan kecepatan akses ke situs web yang sering dikunjungi.Mencegah serangan DNS Spoofing atau DNS Cache Poisoning dengan memastikan bahwa alamat IP yang dikembalikan benar-benar berasal dari server DNS yang sah.

29. Jelaskan keterkaitan antara dns server, web server dan database server


Jawaban:

Keterkaitan diantara ketiganya adalah bahwa DNS, Web server, dan Database server sama-sama dibutuhkan dalam menghosting sebuah website yang dynamic.

Penjelasan:

DNS = Domain Name Server, berfungsi untuk mengubah alamat ip kita menjadi sebuah nama yang kita inginkan dan memudahkan orang lain dalam mengakses website kita.

Web server = Sebuah server perangkat lunak yang berfungsi untuk melayani permintaan(request) klien berupa HTTP atau HTTPS dan akan mengirimkan respon sesuai yang diminta.

Database server = Sebuah program komputer yang menyediakan layanan pengelolaan basis data dan melayani sebuah komputer yang kemudian disalurkan ke komputer lainnya menggunakan klien-server

DNS diperlukan untuk memudahkan klien dalam mengakses website kita. Web server dibutuhkan agar terjadi request dan respon antara klien dan server. Dan database dibutuhkan untuk pengelolaan data di sebuah website kita yang menjadikannya dynamic(berubah-ubah).

• Organisasi-organisasi di luar pemerintah semakin banyak bermunculan

• Kompetisi produk semakin kuat karena masuknya barang impor ke Indonesia  

• Tingkat kebutuhan hidup masyarakat semakin tinggi

3. Bidang sosial dan budaya

• Perkembangan teknologi, informasi, dan transportasi semakin pesat

• Apresiasi terhadap kebudayaan local semakin berkurang

• Semakin lunturnya nilai-nilai keagamaan dalam diri masyarakat

34. Common network services pada server telah ditentukan diantaranya: Web Server, Database Server, DNS Server, dan DHCP Server. Jika diimplementasikan pada SO Linux/Debian maka paket apa saja yang cocok diinstall untuk layanan diatas. Jelaskan!


Penjelasan:

Untuk mengimplementasikan layanan Web Server pada sistem operasi Linux/Debian, Anda dapat menginstall paket Apache. Apache adalah salah satu web server yang paling populer dan banyak digunakan di seluruh dunia.

Setelah database terbuat, barulah Anda bisa mengolahnya. Dari dalam CPanel, Anda dapat mengklik tombol PHPMyAdmin untuk mengedit database. Berikut ini penjelasan cara menggunakan PHPMyAdmin untuk mengolah database yang sudah terbuat:1. Dari CPanel, klik ikon PHPMyAdmin untuk menggunakan PHPMyAdmin.2. Di halaman PHPMyAdmin Access, pilih database yang ingin diolah dengan PHPMyAdmin pada tabel List of current databases.3. Klik link Enter PHPMyAdmin di sebelah kanan nama database untuk membuka PHPMyAdmin untuk database tersebut. 4. Halaman awal PHPMyAdmin seperti berikut: Anda dapat melihat nama database ada di bagian panel kiri dan dalam keadaan awal terlihat tidak ada tabel di dalam database karena di bagian kanan PHPMyAdmin masih kosong. Tabel adalah tempat untuk menyimpan data dalam sistem database relasional. Di PHPMyAdmin, cara membuat tabel baru seperti berikut ini :

1. Klik tab Structure kemudian isikan nama tabel di textbox Name di groupbox Create new table on nama_database.

2. Klik Go, tabel sudah terbuat. Berikutnya adalah mengisikan ! field-field yang akan dibuat di tabel Isi nama field di kolom Field.

3. Isikan tipe field di kolom Type.

4. Tentukan panjang masing-masing field di kolom Length/Values.

5. Pilih storage engine di combobox Storage Engine dengan MylSAM.

6. Klik Save untuk menyimpan pembuatan field di tabel. 

7. Ketika field terbuat, sebenarnya yang tereksekusi adalah perintah sql di sql query. Nama-nama field terletak di bagian bawah perintah sql tersebut. 

8. Tabel yang dibuat sebelumnya belum memiliki indeks. Untuk membuat indeks, klik pada tombol Primary key di kolom Action. Ketika muncul konfirmasi Do you really want to Alter table dan Add Primary Key, klik OK. 

9. Setelah primary key terbuat, ada tulisan A primary key has been added on nama_field. Indeks digunakan antara lain untuk membuat struktur tabel lebih baik dan juga mempercepat pencarian.
